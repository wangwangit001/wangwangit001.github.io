白嫖攻略：零成本打造！拥有数不清的专属邮箱，这个免费方法真实用！

发表于2025-05-12|更新于2025-09-01|教程
|总字数:2k|阅读时长:6分钟|浏览量:

拥有一个个性化的域名邮箱不仅显得更专业，还能更好地管理你的数字身份。今天，我就来手把手教大家如何利用免费的 dpdns.org 域名和强大的 Cloudflare 邮件路由功能，轻松实现这一切。

第一步：申请 dpdns.org 免费域名

首先，我们需要一个属于自己的域名。dpdns.org 提供免费的二级域名服务，对于个人使用来说是个不错的选择。180天续期一次,可以用我的续期网站设置到期通知!

  1. 访问 dpdns.org 域名注册网站： 打开浏览器，前往 https://dash.domain.digitalplat.org。 你可能需要注册一个账户并完成一些验证步骤（例如 GitHub 账号验证或 KYC 认证，具体流程以网站当时的政策为准）。
  2. 查找并注册你心仪的域名： 在域名查找框中输入你想要的前缀，然后选择 .dpdns.org 后缀。如果该域名尚未被注册，你就可以按照网站提示完成注册流程。 注册过程中，可能会要求你填写一些个人信息，并提供 Name Server（域名服务器）地址。 如果你打算后续使用 Cloudflare，这里可以暂时随便填写两个网址，因为之后我们会统一修改为 Cloudflare 的 Name Server。

请注意： 免费域名政策可能随时发生变化，请以 dpdns.org 官方网站的最新说明为准。

第二步：将域名托管到 Cloudflare

Cloudflare 是全球知名的网络服务提供商，其免费套餐已经包含了我们需要的 DNS 托管和邮件路由功能。

  1. 注册 Cloudflare 账户： 如果你还没有 Cloudflare 账户，请前往 https://www.cloudflare.com 注册一个。

  2. 添加你的 dpdns.org 域名到 Cloudflare：

    • 登录 Cloudflare 控制面板后，点击“添加站点”或类似按钮。
    • 输入你刚刚申请到的 你的前缀.dpdns.org 域名，然后点击“添加站点”。
    • Cloudflare 会引导你选择套餐，选择免费套餐即可。
    • 接下来，Cloudflare 会扫描你域名当前的 DNS 记录（如果你在 dpdns.org 注册时填写了其他的 Name Server，这里可能会显示出来）。不用担心，直接点击“继续”。

  3. 更改域名的 Name Server： 这是将域名管理权限交给 Cloudflare 的关键一步。

    • Cloudflare 会提供两个它自己的 Name Server 地址（通常类似于 xxx.ns.cloudflare.com 和 yyy.ns.cloudflare.com）。 务必准确复制这两个地址。
    • 回到你注册 dpdns.org 域名的管理后台（通常是 https://dash.domain.digitalplat.org）。找到管理你域名 Name Server 或 DNS 服务器设置的地方。
    • 将 dpdns.org 默认或你之前填写的 Name Server 地址，修改为你从 Cloudflare 获取到的那两个新的 Name Server 地址。
    • 保存更改。

  4. 等待 DNS 解析生效： Name Server 的更改通常需要一些时间在全球范围内生效，短则几分钟，长则可能需要几个小时。 Cloudflare 会定期检测，当它检测到你的域名已经成功指向其 Name Server 后，你的域名状态在 Cloudflare 控制面板上会显示为“有效”或“Active”。 你也可能会收到 Cloudflare 发送的域名激活成功邮件。

第三步：在 Cloudflare 上开启并配置电子邮件路由

域名成功托管到 Cloudflare 后，我们就可以开始配置邮件路由了。

  1. 登录 Cloudflare 控制面板并选择你的域名。

  2. 找到“电子邮件”或“Email”选项： 在域名管理界面的左侧菜单栏中，你应该能找到“电子邮件”或“Email Routing”的选项。

  3. 启用电子邮件路由功能：

    • 点击进入电子邮件路由功能页面。
    • 如果这是你第一次使用该功能，系统可能会有一些引导说明。点击“Enable Email Routing”（启用电子邮件路由）或类似按钮来激活此功能。 Cloudflare 可能会自动为你添加一些必要的 DNS 记录（例如 MX 记录和 TXT 记录）来确保邮件能够正确路由。

  4. 设置 Catch-all 地址（全域邮箱）： 这是实现无限域名邮箱的关键。

    • 在电子邮件路由的设置界面，找到“Catch-all Address”或“所有邮箱地址”之类的选项。
    • 启用这个功能。
    • 在“转发到”或“Send to”的目标地址栏中，输入你最常用的那个接收邮件的电子邮箱地址（比如你的 Gmail、Outlook 邮箱或其他任何你常用的邮箱）。
    • Cloudflare 会向你填写的这个目标邮箱发送一封验证邮件。
    • 登录你的目标邮箱，找到这封来自 Cloudflare 的验证邮件，并点击邮件中的验证链接以确认该邮箱归你所有。

  5. （可选）跳过添加自定义地址，直接使用 Catch-all： 按照我们开头设定的思路，为了简化操作，你可以直接启用 Catch-all 功能，而无需单独为某个特定的前缀（如 info@你的域名.dpdns.org 或 contact@你的域名.dpdns.org）创建单独的路由规则。 开启 Catch-all 后，任何发送到 任何字符@你的域名.dpdns.org 的邮件，都会被转发到你设置的那个常用邮箱。

第四步：测试你的无限域名邮箱

配置完成后，就可以进行测试了：

  1. 使用你的另一个邮箱账户（或者让朋友帮忙）。

  2. 尝试发送几封邮件到一些你并未在 Cloudflare 中明确设置过，但属于你 dpdns.org 域名下的随机邮箱地址，例如：

    • test123@你的前缀.dpdns.org
    • random-prefix@你的前缀.dpdns.org
    • anything-you-want@你的前缀.dpdns.org

  3. 稍等片刻，然后登录你之前设置为 Catch-all 转发目标的那个常用邮箱。

  4. 检查是否收到了这些测试邮件。如果一切设置正确，所有发送到你 .dpdns.org 域名下任意前缀的邮件，都应该会出现在你的常用邮箱收件箱里。

写在最后

通过以上步骤，你就成功创建了一个属于自己的、基于 dpdns.org 免费域名和 Cloudflare 邮件路由的无限域名邮箱。这样一来，无论你在哪个网站注册服务，都可以使用一个该网站名@你的前缀.dpdns.org这样的专属邮箱，既方便记忆和管理，又能有效保护你的主邮箱地址不被泄露。

一些需要注意的点：

  • 发信功能： Cloudflare 的电子邮件路由主要提供的是邮件 接收和转发 功能。 如果你需要以你的自定义域名邮箱（例如 xxx@你的前缀.dpdns.org）作为发件人来发送邮件，通常需要借助你的目标邮箱服务商（如 Gmail、Outlook）的相关设置（例如“通过Gmail从其他电子邮件地址发送邮件”功能），或者使用其他支持 SMTP 发信的服务。
  • DNS 记录的重要性： Cloudflare 邮件路由依赖于正确的 DNS 记录（主要是 MX 记录和可能的 TXT 记录，用于 SPF 等验证）。 在启用邮件路由时，Cloudflare 通常会自动添加这些记录。请确保这些记录没有被意外删除或修改。
  • 隐私和安全： 虽然 Catch-all 很方便，但也意味着所有发送到该域名的邮件（包括可能的垃圾邮件）都会被转发。你需要依赖你最终接收邮件的邮箱服务商的垃圾邮件过滤功能。
  • 服务条款和政策： 请务必遵守 dpdns.org 和 Cloudflare 的服务条款和使用政策。 免费服务的政策也可能发生变动，建议定期关注其官方通知。

希望这篇详尽的教程能够帮到你！拥有一个无限域名邮箱，就像给你的数字生活加了一个强大的邮件处理中心，让一切井井有条。动手试试看吧，你会发现它并没有想象中那么复杂！
