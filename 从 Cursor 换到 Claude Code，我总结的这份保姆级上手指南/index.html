核心概念：理解Claude Code的工作方式

1. 三种工作模式

编辑模式（Edit Mode）- 默认

这是你最常用的模式，Claude可以：

  • 创建、修改、删除文件
  • 执行命令
  • 运行测试

切换方式： 默认就是这个模式

计划模式（Plan Mode）

在开始大型任务前使用，Claude会：

  • 制定详细的实施计划
  • 列出所有步骤
  • 创建计划文档供参考

切换方式： 按 Shift+Tab

对话模式（Chat Mode）

纯对话，不执行任何操作，适合：

  • 讨论架构设计
  • 理解代码逻辑
  • 获取建议

切换方式： 按 Shift+Tab 两次

CLAUDE.md是Claude每次启动时自动读取的配置文件，就像是给Claude的”工作手册”。

DON’T - 避免的做法

故障排除：常见问题解答

问题1：Claude不执行命令，总是要求确认

解决方案：

  1. 确保项目有Git管理：git init
  2. 使用自动模式启动：claude --dangerously-skip-permissions

问题2：Claude忘记了项目上下文

解决方案：

  1. 检查CLAUDE.md是否存在和更新
  2. 使用 @claude.md 显式引用配置
  3. 在长对话中定期总结关键信息

问题4：检查点回滚失败

解决方案：

  1. 确保没有未提交的Git更改
  2. 使用 /checkpoint list 查看所有检查点
  3. 手动指定检查点：/rewind [checkpoint-id]

问题5：API限制错误

解决方案：

  1. 检查API配额：访问 https://code.claude.com/usage
  2. 优化提示词，减少不必要的对话
  3. 使用任务分解减少单次请求的复杂度
